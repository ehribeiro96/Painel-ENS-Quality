# Patch seguro

Implemente a menor alteracao que resolva o pedido.

Regras do projeto:
- Nao altere backend, frontend, .env ou docker-compose sem pedido explicito.
- Preserve a arquitetura modular existente.
- Nao exponha segredos.
- Nao adicione dependencias.
- Antes de editar, leia os arquivos relacionados.
- Depois de editar, mostre os arquivos alterados e os comandos de validacao.
