# Política de encoding pt-BR

- Todos os arquivos novos devem ser UTF-8.
- Markdown, JSON, YAML, `.ps1` e `.py` devem preservar acentuação.
- PowerShell 7 é preferencial para consistência de UTF-8.
- Evitar `chcp` como solução principal.
- Não usar ANSI no projeto.
- Validar palavras com acento como ação, solução, validação e homologação.
