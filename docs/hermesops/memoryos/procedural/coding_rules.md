# Coding Rules

- every coding task must include objective, stack, plan, validation, rollback, and memory candidate
- keep patches minimal
- validate offline before commit
- do not read secrets or prohibited modding material

## pt-BR
- pt-BR é o idioma padrão de resposta operacional do HermesOps, salvo quando o usuário pedir outro idioma ou quando a sintaxe técnica exigir preservação em inglês.
- Explique em português, mas preserve código, APIs, flags, funções, classes, comandos e paths.
- Mensagens de erro originais devem ser reproduzidas sem tradução.
